# 백트래킹이란?

현재 상태에서 가능한 모든 후보군을 따라 들어가며 탐색하는 알고리즘. 선택지가 몇 가지 나오는 게임을 생각해보면 이해하기 쉽다. (역적재판 등등)

# 백트래킹의 구현(상태공간트리를 생각해보기)

-   각 상태는 전에 어떤 것이 선택되었는지 기억해야 함
-   각 상태마다 선택지가 걸러지고, 나누어짐
-   **순열 및 조합**에 주로 사용

```js
const arr = Array(M).fill(0);
const isUsed = Array(N + 1).fill(0); // 숫자 === 인덱스

function f(k) {
    // base condition: k === M이면 이미 M개를 다 골랐다는 뜻
    if (k === M) {
        console.log(arr.join(" "));
        return;
    }
    for (let i = 1; i <= N; i++) {
        if (!isUsed[i]) {
            arr[k] = i;
            isUsed[i] = 1;
            f(k + 1);
            isUsed[i] = 0;
        }
    }
}
f(0);
```

# 백트래킹 구현 시 주의사항

-   상당한 구현력을 필요로 함
-   실수하기도 쉬움
-   재귀처럼 틀리더라도 원일을 찾기 굉장히 힘듦
-   가지치기로 인해 후보군이 삭제되는 경우가 빈번해지면 시간복잡도를 정확히 정의하기 힘듦
-   하지만, 응용할 거리가 그렇게 많지 않기 때문에 예제를 꼼꼼히 풀다보면 기본 코드 형태도 익히면서 활용하기 괜찮을 것!
